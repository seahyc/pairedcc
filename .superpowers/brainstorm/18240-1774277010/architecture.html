Sub-project 1: Architecture Overview

Markdown editor + Yjs collaboration + basic agent connection

System Architecture

Key flows

Human edits: Browser → Tiptap → Yjs update → WebSocket → Server → broadcast to all peers (other browsers + agents)
Agent edits (CLI): Agent Yjs peer → WebSocket → Server → broadcast to browsers + other agents
Agent edits (MCP): Claude Code calls edit_doc tool → REST API → Server applies Yjs update → broadcast
@-mention: Server detects @agent-name in Yjs update → pushes notification to agent's WebSocket → agent processes and responds
